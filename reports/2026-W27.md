### 3. MCP Registry

- 类型：mcp
- 证据等级：official
- 更新状态：新收录
- 适用阶段：implementation / ops
- 关键来源：[MCP Registry docs](https://modelcontextprotocol.io/registry/about)；[Official Registry](https://registry.modelcontextprotocol.io/)；[GitHub](https://github.com/modelcontextprotocol/registry)；[MCP Blog](https://blog.modelcontextprotocol.io/posts/2025-09-08-mcp-registry-preview/)

**证据要点**：采集时间 2026-07-02T09:43+08:00。MCP 官方文档说明 Registry 是 official centralized metadata repository，提供 REST API、namespace management 与标准安装/配置元数据；GitHub 页面显示 `modelcontextprotocol/registry` 约 6,974 stars、885 forks、92 issues、30 PRs、648 commits，latest release `v1.7.9` 于 2026-05-12 发布；官方 Registry 页面提供 production API base URL。

**项目介绍**：MCP Registry 正在把 MCP server 发现从 awesome 清单推向可消费的官方元数据层。它不是直接托管 server 代码，而是托管指向包、远程 server、安装方式和能力描述的标准化 metadata。

**竞品/替代项**：

- [modelcontextprotocol/servers](https://github.com/modelcontextprotocol/servers)：servers 仓库是参考实现与示例，README 明确建议浏览 Registry 查找已发布 servers。
- [punkpeye/awesome-mcp-servers](https://github.com/punkpeye/awesome-mcp-servers)：awesome 清单覆盖面更广，但可信同步和机器消费应回到官方 Registry/API。

**推荐理由**：对周更雷达来说，Registry 是 MCP 类资产的优先同步源：它有官方背书、API、namespace 验证和安全/反垃圾机制，比单纯抓 awesome 清单更适合自动化。

**最佳使用范式**：用于 implementation 与 ops 阶段：先从 Registry 同步候选 server 元数据，再回到对应 GitHub/包管理器验证代码、license、维护状态；企业内部可按官方 OpenAPI 形态派生私有 registry。
