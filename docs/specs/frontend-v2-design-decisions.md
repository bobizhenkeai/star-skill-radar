# 情报站前端 v2 设计决策简报（架构师裁决）

> 本文件是「前端 v2 首屏信息架构与视觉系统改版」的**权威设计决策**，由架构师综合 5 份调研后裁决。实施窗口以本文的「决策 + 验收标准」为准；**具体实现路径（标记、CSS 数值、函数结构）由实施窗口自主决定**，需要更深的依据时向下钻取对应调研报告。

## 0. 一句话诊断（5 份调研共识）

当前观感问题的根子是**「一堆白盒子浮在纸面上」+ 层级错位**，不是「不够暖」或「卡片不够多」。首屏"今日摘要"是**同权卡片网格**、hero 统计是**3 张独立白卡**——两者都缺主次层级。修复应从**信息层级与结构**入手，而非贴皮。

## 1. 核心方向裁决

**采用「纸面 briefing board + 轻 bento」（B2 方向 A）**，以 A1 的「**总览条 + 分组短列表**」为结构骨架，叠加 B2 的图标徽章/日报头版视觉层、B1 的排版与对比度地基、A3 的 grouped brief band 头部、A2 的可逆披露交互。

保留不变：暖色羊皮纸基调、墨绿 `--accent` 主色、8px 圆角、"耳语阴影"量级、纯静态原生实现、无外部 CDN/字体。

## 2. 两处调研分歧的架构师裁决

**D1 · 概览层要不要"板内展开"（A1 vs A2）**
- A1：概览层应短而稳，超量条目不在板内展开，改为跳转/紧凑全展示。
- A2：若保留展开，必须改成可逆、可访问 toggle。
- **裁决**：首屏概览改为"总览条 + 分组短列表"后，多数分组直接以紧凑列表**全展示**或给"查看本类全部 →"跳转到下方完整卡片区。**硬性底线：彻底消除"单向展开不可收起"（Q1）。** 若实施窗口在某些长分组（如简讯）仍保留板内展开，则**必须**按 A2 规范做成可逆 disclosure（按钮常驻、`aria-expanded`/`aria-controls`、extra panel 在按钮之后、44px 热区、仅图标旋转+轻淡入、`prefers-reduced-motion` 兜底、收起时焦点回收）。两条路径实施窗口择优，但结果都必须可逆。

**D2 · bento 网格 vs 列表（A1 vs B2）**
- **裁决**：不做"所有分组同权三列网格"，也**不做** masonry/`grid-auto-flow: dense`（视觉顺序会与 DOM/键盘顺序脱钩）。以**分组短列表**为主，允许**少量加权/跨栏**（头条所属分组、当日最多分组、简讯长列表）作强调。移动端回到单列日报清单。

## 3. 逐组件决策（映射到现有组件 + 验收标准）

### 3.1 hero 统计区 `.hero-stats`（对应 Q2，依据 A3/B1）
- **决策**：从"3 张独立白卡"改为**一条 grouped brief band**——由父容器承担表面（暖色 surface + 轻描边），子项改为透明分区（细分隔线，不再各自成卡）。
- 指标重排为 **`最新日期`（新鲜度，主指标，独占一行/最大 inline 空间）> `期日报数`（覆盖度）> `累计重点`（规模）**。
- 文案 `已加载重点数` → **`累计重点`**（去实现口吻）。
- 日期值包 `<time datetime="YYYY-MM-DD">`，加 `white-space: nowrap`，保留 `tabular-nums`；**根治点是给日期完整一行空间，不是只加排版属性**。
- 移动端压成"信息带"（主行 + 次级双列/多行），不做 3 张竖卡墙。
- **验收**：视觉上是"纸面上的一条概览带"而非 3 个白盒；`2026-07-02` 桌面/移动均单行不断行不溢出；最新日期视觉权重最高。

### 3.2 总览锚点 `overview-lede`（依据 A1/B2）
- **决策**：把"今日共 N 条 · X 重点 · Y 简讯 · 头条：…"从一句 prose 拆成**日报头版条（masthead）**三层：① 主数字 `N`（最大）② breakdown 小徽标（重点/简讯/类目）③ `头条：…` 独占一行。
- **验收**："今天发生了多少"成为独立视觉锚点，一眼可见；头条不再是句尾附言。

### 3.3 概览主体 `overview-grid` / `overviewCategoryCard`（依据 A1/B2）
- **决策**：从大屏 3 列同权网格改为**分组短列表 / summary sections**；分组排序按 **头条所属分组 → 当日条目最多分组 → 其余按数量降序 + 稳定类型兜底**。
- 每个分组 header：**内联 SVG 图标徽章 + 分类名 + 数量**（数量用 `--mono` 小字暖灰，不与标题同权）。
- 简讯（长列表）从普通分类卡**脱离为紧凑列表/跨栏块**，解决"7 条把单卡拉长"的失衡。
- 类目色只用于图标徽章/3px 细线/浅 chip，**不染整卡**。
- **验收**：桌面无大片空洞、无 1-条卡与 7-条卡同权竞争；分组顺序反映"今天最该先看什么"。

### 3.4 概览条目 `overviewBulletItem` / 摘要 copy（依据 A1/B2）
- **决策**：条目 = **项目名（区分性关键词前置 6–10 字内）+ 一句 briefing copy**。overview 的一句话只取「它是什么 / 为什么重要 / 今天变了什么」**之一**，不并存三层。
- 截断策略：**语义优先（首句/首分句）+ CSS `line-clamp` 两行兜底 + 必要时省略号**，替代当前纯字符 `truncateText(...,46)`；目标约 **18–36 个中文字符**（工程启发，非硬标准）。
- **边界（重要）**：本轮仅做**客户端**语义截断，数据仍取现有 `summary`/`recommendation` 字段。"为 overview 单独生成 briefing copy 字段"属于**数据契约变更**，不在本前端窗口范围（见 §5）。
- **验收**：移动端摘要 1–2 行、桌面 ≤2 行；不出现机械断句的"半句"。

### 3.5 Q1 折叠 `overviewMoreButton`（对应 Q1，依据 A2）
- **决策**：见 §2 D1。硬性底线——**消除单向不可逆展开**。保留展开时须为可逆 disclosure；`evidencePanel` 的原生 `<details>` 保持不变（已是正确实现）。
- **验收**：可重复"展开→收起→再展开"；键盘 `Tab`+`Enter`/`Space` 可操作；屏幕阅读器可知 expanded/collapsed；折叠后 extra 内链不可 `Tab`；按钮移动端 ≥44×44px；`prefers-reduced-motion` 下无非必要动画；展开/收起后按钮不被自身内容顶走。

### 3.6 排版系统（依据 B1）
- **决策**：正文/卡片说明/摘要移动端**≥16px**、行高 `1.56–1.68`；`12–14px` 仅限 chip/时间戳/短标签。长正文段落 `max-inline-size: 38em`（中文约 36–40 字/行）。字号阶梯与字体栈按 B1 §4；`--sans` 系统级中文优先、`--serif` 仅大标题、`--mono` 仅数字/日期/短标识。
- **验收**：移动端无正文停留在 13–15px；长中文段不超约 40 字/行。

### 3.7 配色与对比度 token（依据 B1）
- **决策**（在保暖色基调前提下修复对比度隐患）：
  - `--paper-strong` `#ffffff` → `#fffdf8`（或 `#fdfbf6`），不破坏暖纸系统。
  - 新增/使用 `--line-ui`（约 `#948978`，≥3:1）承担**必须识别的控件/边界**；装饰发丝线继续 `--line`。
  - `--signal` 不直接用于普通/小号文本；红色文本用深红 token（如 `#a9371f`）。
  - `--cat-subagent` chip 文本加深（如 `#874c2f`）以达普通文本 AA。
  - 类目色低饱和、小面积、只服务分类识别，不发展成第二品牌色。
- **验收**：正文 ≥4.5:1、大字/有意义图形 ≥3:1；关键色对复算通过。

### 3.8 图标体系（依据 B2）
- **决策**：内联 SVG，`viewBox="0 0 24 24"`、`fill="none"`、`stroke="currentColor"`、`stroke-width≈1.8–2`；装饰性图标 `aria-hidden="true"`，纯图标按钮加 `aria-label`。**不用 emoji、不用 icon font、不接 CDN/图标包**。可参考 Lucide/Heroicons 风格并内联必要 SVG（遵守许可证）。分类→图标语义映射见 B2 §图标体系（实施窗口可微调具体字形）。
- **验收**：分类识别度提升；无外部图标依赖；图标对比/可访问性达标。

## 4. 反模式清单（验收 checklist，任一命中需修正）

masonry/`dense` 补洞；1 条与 7 条分组同权卡；单向不可逆展开；关键内容压在右侧第三列；纯字符截断出"半句"；标题共享冗长前缀；交错 zigzag 排布；大面积类目色做主层级；hero KPI 用实现口吻（"已加载"）或虚荣指标/趋势箭头装饰；纯白卡/重阴影/大圆角；日期加省略号或用地区歧义格式；移动端 3 张竖卡墙；引入外部字体/CDN/图标字体。

## 5. 明确的范围边界（本前端窗口不得触碰）

- **只改** `site/`（`index.html` / `app.js` / `styles.css` + 内联 SVG 资源）。
- **不改** `data/`、`reports/`、`docs/prds/`、`docs/specs/`（除本文件外）、以及数据契约字段与 SOP。
- overview 的"独立 briefing copy 字段"、RSS/订阅、历史资产总表、分享/品牌收口（OG/favicon）**均不在本轮**，后续单独排期。
- 不得破坏现有能力：日期路由 `#/date/…`、type/stage 筛选、月度归档、摘要看板跳转、证据 `<details>` 折叠、无 JS 降级、既有安全处理（`isSafeHref` 等）。

## 6. 建议实施顺序（分阶段提交，便于回归审查；顺序非强制）

1. **P0 修复 + 头部**：hero grouped brief band（Q2）+ Q1 可逆披露/去板内展开。
2. **P1 结构**：overview masthead 总览条 + 分组短列表 + 排序 + 简讯脱离。
3. **P2 视觉皮层**：图标体系 + 类目色收敛 + 质感细节。
4. **P3 地基**：排版字号/行高/行长 token + 对比度 token 收口（可与 P2 合并）。

## 7. 验证要求（AGENTS.md 一致）

本地起静态服务器（`python -m http.server`，勿 `file://` 直开）+ chrome-devtools MCP 桌面/移动视口（移动端用 `emulate`，勿 `resize_page`）；核对：Q1 可逆、Q2 不断行、筛选/路由/归档/证据折叠无回归、控制台无报错、桌面与移动渲染正常、关键对比度达标。
