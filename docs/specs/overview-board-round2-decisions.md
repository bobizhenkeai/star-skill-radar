# overview 看板 Round 2 — 站点线设计决策与验收契约

> 架构师产出的**上下文材料 + 产出契约**。诊断与验收标准为准；**实现路径由实施窗口自主选择**。本轮为站点线，仅动 `site/app.js` + `site/styles.css`。数据/契约侧（简讯 `gist` 生成与回填）由数据/SOP 线按 `docs/specs/data-contract.md` v2.3 并行处理，本线只负责**消费**。

## 背景

Phase 8 上线后用户线上实测，反馈四项（Q1、Q2 + 两个"顺便"）。本文固化站点线四项的根因与验收标准。

## Q1 — 类目卡展开后"收起"按钮卡在中间（割裂感）

**根因**：Phase 8 只把**简讯卡**改成了"单连续列表 + 按钮作为最后一项"的模式；**类目卡仍走旧 `overviewDisclosure`**（结构为：基准若干条 → 按钮 → 展开的其余条目 panel），所以展开后按钮夹在基准条目与展开条目之间。

**验收契约**：
1. 类目卡展开后，"收起，回到前 N 条"按钮必须位于**该卡所有可见条目的最下方**，不得夹在中间。
2. 折叠/展开保持**可逆**；`aria-expanded`/`aria-controls`/键盘 Enter·Space/收起时焦点回到按钮/`prefers-reduced-motion` 兜底**不得回退**。
3. 收起文案沿用**动态基准数量**（不得写死）。
4. 类目卡为单列，本项不涉及奇偶占位；但**不得**引入内部空洞或回归 Phase 8 的简讯修复。
5. 建议与简讯卡**统一为同一套 disclosure 结构/助手**（减少两套逻辑漂移），但具体如何统一由实施窗口决定。

## Q2 — 简讯看板文案被切割（消费简讯 `gist`）

**根因**：简讯在两处渲染——看板 `overviewBriefBulletItem` 用 `one_liner` 且套 **2 行 `line-clamp` 截断**（无 `is-gist`）；详情 `briefSection` 用完整 `one_liner`。`one_liner` 含"描述；证据(stars…)"，看板 2 行放不下即从中间截断加"…"。highlights 已用 `gist`（不截断）解决同类问题，简讯此前无对应字段。

**契约依据**：`data-contract.md` v2.3 已为 briefs 新增可选 `gist`（自洽短句、无证据、看板不截断），`one_liner` 保留为详情正文。

**验收契约**（站点消费侧）：
1. **overview 简讯概览文案**：有 `brief.gist` 用 `gist` 且**不截断**（与 highlight `gist` 的 `is-gist` 无截断表现一致）；缺失则降级为对 `one_liner` 的**语义提炼**（提炼前尽量剥离证据尾巴，保证读到的是完整描述而非中途"…"）。
2. **详情「简讯」区**（`briefSection`）文案**保持使用完整 `one_liner`**，不受 `gist` 影响。
3. **向前/向后兼容**：`gist` 缺失、为空、或为非字符串脏值时均不得报错、不得出现 `[object Object]`（沿用 highlights 的 `gist` 归一化/类型守卫思路）。
4. 该文案与 Q1 的可逆展开、Phase 8 的无空洞占位**不得互相破坏**。

## 顺便①：跨文件耦合注释

`OVERVIEW_BRIEF_DESKTOP_COLUMNS = 2` 与 CSS `@media (min-width: 640px)` 下简讯网格 `repeat(2, …)` 是**跨文件奇偶耦合**。

**验收契约**：在该常量旁加一行注释，指明其必须与 CSS 640px 处的简讯列数保持一致（改 CSS 列数须同步改此常量）。仅注释，无行为变更。

## 顺便②：类目卡等高拉伸底部留白

**根因**：`.overview-grid` 默认 `align-items: stretch`，同一行内条目数悬殊的两张类目卡会被拉到等高，矮卡底部出现空白。

**验收契约**：
1. 同行类目卡因内容多寡不同而产生的**底部大面积空白**应消除或显著缓解（如让卡片贴合自身内容高度）。
2. 需在"卡片贴合内容（行高参差）"与"行高一致（底部留空）"之间做出取舍并使整体观感协调；具体策略（`align-items`、内容高度均衡等）由实施窗口决定。
3. 不得破坏移动端单列布局、不得回归简讯卡与 Q1/Q2 的表现。

## 通用约束

- 仅 `site/app.js` + `site/styles.css`；不改数据/契约（那是并行数据线的事）。
- 不影响 gist（highlights）、`#/catalog` 资产总表、OG/favicon、类目跳转、筛选、路由等既有功能。

## 验证清单（完成后须自测并回报）

- `node --check site/app.js` 通过。
- 本地静态服务器桌面（≥640px 2 列）+ 移动（<640px 1 列）视口目视：
  - **Q1**：类目卡（取一个 >3 条的类目）展开后按钮在最底部、可逆、文案数量正确、键盘/焦点/reduced-motion 正常。
  - **Q2**：有 `gist` 的简讯在看板**完整无"…"**；无 `gist`（临时造 fixture）时降级为 `one_liner` 完整提炼、无中途截断；详情简讯区仍显示完整 `one_liner`；脏 `gist` 不报错。
  - **顺便②**：同行类目卡无异常底部大留白；简讯卡 Phase 8 的 3/4/5/7 条无空洞仍成立。
  - 既有功能（catalog、筛选、路由、OG/favicon、highlight gist）无回归；控制台无报错。
